# Contract: `RenderBlocks` dispatcher

**File**: `src/components/sections/RenderBlocks.tsx` (+ `src/components/sections/registry.ts`)

**Cited from**: `docs/BLOCK_LIBRARY.md` §8.

## Signature

```typescript
import type { Page } from '@/payload-types'

export interface RenderBlocksProps {
  blocks: Page['layout'] | null | undefined
}

export function RenderBlocks(props: RenderBlocksProps): React.ReactElement
```

## Behaviour

1. If `blocks` is `null`, `undefined`, or empty array → returns an empty fragment (no DOM, no warning).
2. For each block in `blocks`:
   - Look up `registry[block.blockType]`.
   - If found: render `<Component key={block.id ?? index} {...block} />`.
   - If not found and `process.env.NODE_ENV !== 'production'`: `console.warn` once per unknown blockType per render (`Unknown blockType: ${blockType}`), return `null` for that slot.
   - If not found and production: silently skip.
3. Each component receives the full block object as props — no per-call data shaping.

## Registry contract

```typescript
// src/components/sections/registry.ts
import type { ComponentType } from 'react'

export const registry: Record<string, ComponentType<any>> = {
  hero: Hero,
  'case-study-hero': CaseStudyHero,
  content: ContentBlock,
  // … one entry per layout block in src/payload/blocks/layout/index.ts
}
```

**Invariant**: every block registered in `src/payload/blocks/layout/index.ts` MUST have a registry entry. Enforced by a Vitest test that iterates the block-config exports and asserts `registry[block.slug] !== undefined`.

## Test contract

- `tests/int/render/renderBlocksUnknownType.test.ts`:
  - Empty array → no DOM, no warning.
  - Known block → renders the registered component with full props.
  - Unknown blockType in dev → renders nothing for that slot, emits a single warning.
  - Unknown blockType in prod → renders nothing, no warning.
- `tests/int/render/registryCoverage.test.ts`:
  - Iterates layout-block exports and asserts a registry entry exists for each.

## Stability

- The `Page['layout']` type comes from `payload-types.ts` (auto-regenerated by `npm run generate:types`); changes to the block library propagate automatically.
- Adding a new block requires: (a) new file under `src/payload/blocks/layout/`, (b) register in `index.ts`, (c) new renderer component, (d) new entry in `registry.ts`, (e) regenerate types and importMap. The coverage test enforces all five steps.
